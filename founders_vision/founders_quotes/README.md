## What this folder is for

Topical collections of Thomas's (and other named contributors') writing and spoken words, organized by subject rather than date. Per Thomas: "a folder with files that record my writing on the various topics."

This folder is one of three complementary components in `founders_vision/`. It is **distinct from**:

- **`founders_vision/founders_vision.md`** — the *curated thematic summary*. A single integrated narrative organized by theological concept, written by Thomas (with Claude) and revised in place as understanding matures.
- **`founders_vision/YYMMDD_*.md`** — the *dated seed archive*. Immutable chronological record of what was said on a given day, preserving voice and context.

The topical quotes folder is the **topical retrieval layer**. Where the summary provides integrated narrative and the seed archive provides the raw chronological record, the quotes folder provides Thomas's actual verbal material organized by subject — so that a reader, a future collaborator, or a Christos AI module can look up what Thomas has said about (say) constitutional originalism, or theodicy, or the cosmology of CPP, without reading the full summary or sifting the chronological archive.

## Open conventions

The following conventions are provisional — to be confirmed, adjusted, or replaced by Thomas:

- **Filename:** `topic_slug.md`, lowercase, underscores. No date prefix (topical, not chronological).
- **Each file:** header metadata (contributor, topic summary, sources drawn from), followed by a sequence of dated quotes, each with date, source (spoken / written / fellowship meeting / email / essay), and verbatim text. Quotes inside a file are ordered chronologically within the topic.
- **Attribution:** every quote carries the date it was spoken or written and the source context. When a quote appears in the dated seed archive, the quotes file cites the seed entry.
- **Selection criterion:** a quote is added to this folder when it captures a thought clearly enough to stand alone and be useful for retrieval — typically because it states a principle, answers an objection, or defines a term with precision.
